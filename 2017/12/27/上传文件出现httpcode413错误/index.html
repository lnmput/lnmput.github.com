上传文件出现httpcode413错误

发表于 2017-12-27 更新于 2018-03-27
世人谓我恋长安，其实只恋长安某。
在使用curl上传POST一段数据时，被提示413 Request Entity Too Large，应该是nginx限制了上传数据的大小。 解决方法就是: 打开nginx主配置文件`nginx.conf`，一般在`/usr/local/nginx/conf/nginx.conf`这个位置，找到`http{}`段，修改或者添加 然后重启nginx， 即可。 要是以php运行的话，这个大小`client_max_body_size`要和`php.ini`中的如下值的最大值差不多或者稍大，这样就不会因为提交数据大小不一致出现错误。
